# test-strategy-advisor

测试策略建议 agent。

## 适用场景

- 测试阶段开始前
- 分析代码变更以确定测试优先级
- 推荐测试方法

## 钩子触发点

`pre_stage_TESTING`

## 能力

1. **变更分析**：识别高风险变更
2. **覆盖建议**：确定优先测试区域
3. **测试类型选择**：单元/集成/端到端

## 输出

测试计划，包含：

- 优先区域
- 推荐测试类型
- 覆盖目标

## 配置选项

```yaml
config:
  analyze_coverage: true
  prioritize_areas: true
```

## 调用示例

```
AI: 启动 test-strategy-advisor...

🧪 测试策略建议：

高优先级区域：
1. uploadHandler.ts - 新的文件上传逻辑
2. imageProcessor.ts - 图片处理

推荐测试：
┌──────────────┬────────┬──────────────────────┐
│ 类型         │ 数量   │ 重点                 │
├──────────────┼────────┼──────────────────────┤
│ 单元测试     │ 8      │ 核心函数             │
│ 集成测试     │ 3      │ API 端点             │
│ 端到端       │ 1      │ 上传流程             │
└──────────────┴────────┴──────────────────────┘

覆盖目标：80%
```
