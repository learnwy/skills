# mvp-freeze-architect

MVP 范围定义与功能冻结 agent。

## 适用场景

- PRD 精炼完成后
- 将范围压缩为最小可交付循环
- 对功能应用三态规则

## 钩子触发点

`post_stage_ANALYZING`

## 能力

1. **三态规则**：每个功能必须是：已完成 / 未开始 / 已删除
2. **黑名单检查**：拒绝破坏 MVP 的功能
3. **范围冻结**：锁定 V1 范围，延期其他所有

## 输出

MVP 冻结规格，包含：

- V1 功能列表（已锁定）
- 冻结/延期功能
- 风险驱动的裁剪

## 配置选项

```yaml
config:
  freeze_mode: "v1_only"
  output: "mvp_scope"
```

## 调用示例

```
AI: 启动 mvp-freeze-architect 定义 MVP...

🧊 MVP 冻结结果：

V1 范围（已锁定）：
┌────┬─────────────────────┬────────┐
│ #  │ 功能                │ 状态   │
├────┼─────────────────────┼────────┤
│ 1  │ 文件上传            │ ✅ V1  │
│ 2  │ 图片预览            │ ✅ V1  │

❄️ 已冻结（不在 V1）：

预估 V1 时间：4 小时
```
