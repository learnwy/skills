# code-reviewer

综合代码审查 agent。

## 适用场景

- 实现完成后
- 测试阶段开始前
- 检查代码质量和安全性

## 钩子触发点

`post_stage_IMPLEMENTING`

## 能力

1. **缺陷检测**：发现潜在 bug
2. **安全扫描**：识别安全漏洞
3. **代码质量**：检查风格、复杂度
4. **最佳实践**：验证设计模式

## 输出

代码审查报告，包含：

- 按严重程度分类的问题
- 改进建议
- 已审查文件

## 配置选项

```yaml
config:
  review_type: "comprehensive"
  check_security: true
```

## 调用示例

```
AI: 启动 code-reviewer...

🔍 代码审查结果：

已审查文件：5

❌ 关键（0）

⚠️ 高（1）：
- uploadHandler.ts:45 缺少输入验证

💡 中（2）：
- 建议提取魔术数字（第 23 行）
- 函数过长，建议拆分（第 78-120 行）

✅ 低（3）：
- 建议小幅命名改进

总体：需修复 1 个问题后即可进入测试
```
