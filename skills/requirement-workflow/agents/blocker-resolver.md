# blocker-resolver

阻塞问题分析与解决 agent。

## 适用场景

- 工作流进入 BLOCKED 状态
- 需要识别根本原因
- 建议变通方案

## 钩子触发点

`on_blocked`

## 能力

1. **根因分析**：识别阻塞原因
2. **变通方案建议**：提出替代方案
3. **升级路径**：推荐后续步骤

## 输出

解决方案计划，包含：

- 阻塞详情
- 建议操作
- 变通方案

## 配置选项

```yaml
config:
  suggest_workarounds: true
```

## 调用示例

```
AI: 启动 blocker-resolver...

🚧 阻塞分析：

阻塞原因：B 团队 API 文档缺失
类型：外部依赖
严重程度：高

建议操作：
1. ✉️ 向 B 团队负责人升级
2. 📅 安排同步会议
3. 🔄 暂时使用 Mock API（变通方案）

变通方案可用：是
- 基于现有模式创建 Mock
- 基于假设继续开发
- 真实 API 可用后进行验证
```
