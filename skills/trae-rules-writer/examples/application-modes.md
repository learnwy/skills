# 应用模式示例

每种规则应用模式的完整示例。

## 示例 1：始终应用（安全规则）

**场景**：在所有代码中强制安全实践。

### 步骤 1：创建规则文件

```bash
# 创建：.trae/rules/security.md
```

### 步骤 2：编写规则

```markdown
---
alwaysApply: true
---

# 安全实践

## 敏感数据
- 绝不记录密码、令牌、API 密钥或个人信息
- 使用环境变量存储密钥
- 在错误消息中遮盖敏感数据

## 输入验证
- 在入口处验证所有用户输入
- 使用允许列表优于拒绝列表
- 在数据库操作前净化数据

## 认证
- 使用安全的会话管理
- 实现适当的 CSRF 防护
- 强制强密码策略
```

### 步骤 3：验证

开启新聊天并编写任意代码——安全规则将自动生效。

## 示例 2：文件特定（TypeScript 规则）

**场景**：仅适用于 `.ts` 和 `.tsx` 文件的 TypeScript 指南。

### 步骤 1：创建规则文件

```bash
# 创建：.trae/rules/typescript.md
```

### 步骤 2：编写规则

```markdown
---
globs: *.ts,*.tsx
alwaysApply: false
---

# TypeScript 指南

## 类型安全
- 在 tsconfig 中启用严格模式
- 避免 `any`——对未知类型使用 `unknown`
- 当类型显而易见时优先使用类型推断
- 使用可辨识联合处理复杂状态

## 模式
- 类型与实现一起导出
- 使用 `as const` 处理字面量类型
- 对象使用接口（interface），联合使用类型别名（type）

## 导入
- 使用路径别名的绝对导入
- 分组：外部 → 内部 → 相对
```

### 步骤 3：用法

编辑 TypeScript 文件时规则自动激活：

```
用户："创建一个用户服务"
→ 涉及 user.service.ts
→ TypeScript 规则自动激活
```

## 示例 3：智能应用（测试规则）

**场景**：编写测试时应用测试指南。

### 步骤 1：创建规则文件

```bash
# 创建：.trae/rules/testing.md
```

### 步骤 2：编写规则

```markdown
---
description: 编写、审查或讨论测试代码时应用
alwaysApply: false
---

# 测试指南

## 测试结构
- 遵循 AAA 模式：Arrange（准备）、Act（执行）、Assert（断言）
- 每个测试一个断言概念
- 使用描述性测试名称

## 命名约定
- 格式：`describe('[单元]', () => it('当 [条件] 时应 [行为]'))`
- 示例：`it('should return null when user not found')`

## Mock 处理
- 仅 mock 外部依赖
- 在测试之间重置 mock
- 使用依赖注入提高可测试性

## 覆盖率
- 业务逻辑目标 80% 覆盖率
- 工具函数 100% 覆盖率
- 不要测试实现细节
```

### 步骤 3：用法

AI 判断相关性：

```
用户："帮我为认证模块编写测试"
→ AI 检测到测试意图
→ 测试规则自动激活

用户："重构认证模块"
→ 未检测到测试意图
→ 测试规则未激活
```

## 示例 4：手动应用（实验性规则）

**场景**：仅在明确请求时使用的实验性模式。

### 步骤 1：创建规则文件

```bash
# 创建：.trae/rules/experimental-patterns.md
```

### 步骤 2：编写规则

```markdown
---
alwaysApply: false
---

# 实验性模式

## 功能开关
- 用功能开关包装实验性功能
- 使用 LaunchDarkly SDK 管理开关
- 生产环境中所有开关默认为 false

## A/B 测试
- 使用分析事件追踪转化
- 在会话开始时实现变体分配
- 记录所有变体分配

## 渐进式发布
- 从 5% 流量开始
- 增量前监控错误率
- 准备好熔断开关
```

### 步骤 3：用法

必须显式引用：

```
用户："使用 #experimental-patterns，实现新的结账流程"
→ 实验性规则通过引用激活
```

## 快速参考

| 模式 | 配置 | 激活方式 |
| ---- | ---- | ------- |
| 始终应用 | `alwaysApply: true` | 项目中每次聊天 |
| 文件特定 | `globs: *.ts` + `alwaysApply: false` | 当匹配文件参与时 |
| 智能 | `description: "..."` + `alwaysApply: false` | AI 判断相关性 |
| 手动 | `alwaysApply: false`（无 globs/desc） | 聊天中使用 `#RuleName` |

```

       ↓
[根据范围选择模式]
       ↓
┌─────────────────────────────────────────┐
│ 全局强制？      → 始终应用              │
│ 特定文件类型？  → 文件特定              │

│ 仅按需？        → 手动                  │
└─────────────────────────────────────────┘
       ↓
[创建 .trae/rules/<name>.md]
       ↓
[配置 frontmatter]
       ↓
[编写规则内容]
       ↓
[开启新聊天验证]
```
