# Security Rules

## Password Protection
- Never log passwords, API keys, tokens, or secrets in any output
- Use placeholder values like `***` or `[REDACTED]` when displaying sensitive data

## Input Validation
- Always validate input at entry points (API endpoints, form handlers, CLI arguments)
- Reject invalid input early with clear error messages
- Sanitize before processing or storing

## Secrets Management
- Store secrets in environment variables, never hardcode
- Use .env files with .gitignore, never commit secrets

## Error Handling
- Never expose stack traces or internal paths in production errors
- Log security events (failed auth, suspicious input) with context
