# TypeScript Interface Usage

- Always use `interface` instead of `type` for object type definitions
- Use `interface` for public API types and data models
- Use `type` only for unions, intersections, and primitive type aliases
- Example: `interface User { name: string; age: number; }` instead of `type User = { name: string; age: number; }`
